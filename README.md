# 🛠️ 정보처리산업기사 실기 핵심 요약본  
> **V-모델부터 보안, DB, 프로그래밍까지 — 시험 필수 암기 정리**

---

## 🧩 1️⃣ 소프트웨어 개발 생명주기 및 테스트

### 🔹 1.1 V-모델 (Verification & Validation)

| 단계 (좌) | 검증 활동 | 테스트 단계 (우) | 테스트 유형 |
|------------|------------|------------------|---------------|
| 요구사항 | 요구사항 명세 검토 | 인수 테스트 | 사용자 승인 (알파/베타) |
| 분석 | 기능 명세 분석 | 시스템 테스트 | 전체 기능/성능/장애 처리 검증 |
| 설계 | 시스템 구조 검토 | 통합 테스트 | 모듈 결합 검증 (상향식/하향식/빅뱅) |
| 구현 | 코드 검토 | 단위 테스트 | 모듈별 기능 검증 |

---

### 🔹 1.2 테스트 기법 및 분류

| 분류 | 특징 | 주요 기법 |
|------|------|-----------|
| 블랙박스 | 사용자 요구 명세 기반, 외부 동작 검증 | 동치분할, 경계값 분석, 원인-효과 그래프 |
| 화이트박스 | 내부 구조/논리 기반, 내부 동작 검증 | 구문, 조건, 분기, 루프, 기본 경로 검사 |
| 정적 테스트 | 실행 X, 코드/명세 검토 | 워크스루(검토), 인스펙션(심사) |
| 동적 테스트 | 실행 O, 실제 동작 검증 | 블랙박스/화이트박스 |

#### 테스트 목적별 분류
- **회복(Recovery) 테스트**: 결함 발생 후 복구 여부 확인  
- **강도(Stress) 테스트**: 과부하 상태에서도 정상 동작 여부  
- **회귀(Regression) 테스트**: 코드 수정 후 영향 확인  
- **병행(Parallel) 테스트**: 신/구 시스템 결과 비교

## 🧱 2️⃣ 자료구조 & 알고리즘

### 🔹 2.1 기본 자료구조

| 구조 | 특징 | 주요 연산 |
|------|------|------------|
| 스택 (Stack) | LIFO (후입선출) | PUSH, POP |
| 큐 (Queue) | FIFO (선입선출) | ENQUEUE, DEQUEUE |
| 덱 (Deque) | 양쪽에서 삽입/삭제 가능 | — |
| 트리 (Tree) | 계층적 구조 (노드-간선) | 전위, 중위, 후위 순회 |
| 그래프 (Graph) | 정점과 간선의 집합 | BFS(큐), DFS(스택) |
| 해시 테이블 | 키를 해시함수로 변환하여 저장 | 충돌해결(체이닝, 개방주소법) |

### 🔹 2.2 주요 정렬 알고리즘

| 알고리즘 | 특징 |
|----------|------|
| 선택 정렬 | 가장 작은(큰) 값 찾아 교환 |
| 삽입 정렬 | 정렬된 부분에 새 요소 삽입 |
| 버블 정렬 | 인접 요소 비교·교환 |
| 퀵 정렬 | **피벗(Pivot)** 기준 분할 (가장 빠름) |
| 힙 정렬 | 힙(Heap) 구조 이용, O(n log n) |

## 💻 3️⃣ 프로그래밍 & 디자인 패턴

### 🔹 3.1 GoF 핵심 8대 디자인 패턴

| 패턴 | 유형 | 설명 |
|------|------|------|
| 싱글톤 (Singleton) | 생성 | 인스턴스 1개만 생성, 전역 접근 |
| 팩토리 (Factory) | 생성 | 객체 생성 로직을 캡슐화 |
| 옵저버 (Observer) | 행동 | 상태 변화 시 자동 알림 |
| 전략 (Strategy) | 행동 | 알고리즘을 교체 가능하게 캡슐화 |
| 템플릿 메서드 (Template Method) | 행동 | 알고리즘 뼈대 정의, 세부는 하위 클래스 구현 |
| 데코레이터 (Decorator) | 구조 | 동적으로 기능 추가/확장 |
| 프록시 (Proxy) | 구조 | 접근 제어 및 대리자 역할 수행 |
| 컴포지트 (Composite) | 구조 | 트리 구조로 구성, 부분-전체 동일 처리 |

## ⚙️ 4️⃣ 운영체제 (Operating System)

### 🔹 4.1 CPU 스케줄링

| 방식 | 특징 |
|------|------|
| FCFS | 비선점, 선입선출 |
| SJF | 비선점, 짧은 작업 우선 |
| RR | 선점, 시분할, 일정 타임 슬라이스 |
| SRT | 선점, 남은 시간 짧은 순서 우선 |
| 우선순위 | 우선순위 높은 프로세스 먼저 실행 |

---

### 🔹 4.2 교착상태 (Deadlock) 4대 조건

1. **상호배제(Mutual Exclusion)** — 자원을 한 번에 한 프로세스만 사용  
2. **점유 및 대기(Hold and Wait)** — 자원을 점유한 채 다른 자원 대기  
3. **비선점(No Preemption)** — 자원을 강제로 회수 불가  
4. **순환대기(Circular Wait)** — 프로세스 간 순환 대기 발생  

---

### 🔹 4.3 메모리 관리 (페이지 교체)

| 기법 | 설명 |
|------|------|
| FIFO | 가장 먼저 들어온 페이지 교체 |
| LRU | 가장 오래 사용되지 않은 페이지 교체 |
| LFU | 참조 횟수 가장 적은 페이지 교체 |
| OPT | 앞으로 가장 늦게 사용할 페이지 교체 (이론적 최적) |

## 🌐 5️⃣ 네트워크 (Network)

### 🔹 5.1 TCP/IP 4계층

| 계층 | 주요 프로토콜 | 설명 |
|------|----------------|------|
| 응용 | HTTP, FTP, SMTP, DNS, TELNET | 사용자 서비스 |
| 전송 | TCP, UDP | 종단 간 전송 (TCP: 신뢰성 / UDP: 실시간) |
| 인터넷 | IP, ARP, RARP, ICMP | 라우팅 및 주소 관리 |
| 네트워크 액세스 | LAN, WAN, Ethernet | 물리적 전송, 프레임 처리 |

---

### 🔹 5.2 주요 포트번호

| 포트 | 프로토콜 | 설명 |
|------|-----------|------|
| 21 | FTP | 파일 전송 |
| 22 | SSH | 보안 원격 접속 |
| 23 | TELNET | 원격 접속 (비보안) |
| 25 | SMTP | 메일 송신 |
| 53 | DNS | 도메인 이름 해석 |
| 80 | HTTP | 웹 서비스 |
| 110 | POP3 | 메일 수신 (로컬 저장) |
| 143 | IMAP | 메일 수신 (동기화) |
| 161 / 162 | SNMP | 네트워크 관리 |
| 443 | HTTPS | 보안 웹 서비스 (SSL/TLS) |

---

### 🔹 5.3 핵심 프로토콜
- **NAT**: 사설 ↔ 공인 IP 변환  
- **ARP**: IP → MAC 주소 변환  
- **RARP**: MAC → IP 주소 변환  
- **RIP**: 최단 경로 탐색 (Bellman-Ford, 최대 홉 15)

## 🔐 6️⃣ 정보보안 (Security)

### 🔹 6.1 보안 3대 요소 (CIA Triad)

| 약어 | 영어 | 한글 | 설명 |
|------|------|------|------|
| C | Confidentiality | 기밀성 | 허가되지 않은 접근 차단 |
| I | Integrity | 무결성 | 데이터 변조 방지 |
| A | Availability | 가용성 | 정보/서비스 접근 보장 |

---

### 🔹 6.2 암호화 방식

| 분류 | 설명 | 예시 |
|------|------|------|
| 대칭키 | 암/복호화 동일 키 사용 | DES, AES, SEED, ARIA |
| 비대칭키 | 공개키-개인키 쌍 사용 | RSA, ECC |
| 해시(Hash) | 단방향 암호, 무결성 검증 | SHA, MD5 |

---

### 🔹 6.3 클라우드 보안 서비스

| 약어 | 의미 | 설명 |
|------|------|------|
| PaaS | Platform as a Service | 애플리케이션 개발 플랫폼 제공 |
| LaaS | Logging as a Service | 로그 수집/분석 서비스 |
| DaaS | Data as a Service | 데이터 접근 및 분석 |
| SaaS | Security as a Service | 보안 기능을 클라우드로 제공 |

## 🧮 7️⃣ 데이터베이스 (Database)

### 🔹 7.1 SQL 명령어 분류

| 분류 | 명령어 | 설명 |
|------|----------|------|
| DDL | CREATE, ALTER, DROP | 데이터 구조 정의 |
| DML | SELECT, INSERT, UPDATE, DELETE | 데이터 조작 |
| DCL | GRANT, REVOKE | 권한 부여/회수 |
| TCL | COMMIT, ROLLBACK | 트랜잭션 제어 |

---

### 🔹 7.2 트랜잭션 특징 (ACID)

| 특성 | 의미 | 설명 |
|------|------|------|
| A | 원자성 (Atomicity) | 전부 수행 or 전혀 수행 안됨 |
| C | 일관성 (Consistency) | DB의 무결성 유지 |
| I | 독립성 (Isolation) | 병행 실행 시 간섭 없음 |
| D | 지속성 (Durability) | 성공 결과는 영구 반영 |

---

### 🔹 7.3 정규화 단계

| 단계 | 목표 | 제거 대상 |
|------|------|-----------|
| 1NF | 속성 값 원자화 | 반복 속성 |
| 2NF | 부분 함수 종속 제거 | 부분 종속 |
| 3NF | 이행 함수 종속 제거 | 이행 종속 |
| BCNF | 모든 결정자 후보키화 | 비후보키 결정자 |
| 4NF | 다치 종속 제거 | 다치 종속 |

---

### 🔹 7.4 데이터 독립성과 스키마

| 구분 | 설명 |
|------|------|
| 외부 스키마 | 사용자 관점 |
| 개념 스키마 | 전체 논리적 구조 |
| 내부 스키마 | 물리적 저장 구조 |
| 논리적 독립성 | 개념 변경 시 외부 영향 없음 |
| 물리적 독립성 | 내부 변경 시 개념 영향 없음 |
